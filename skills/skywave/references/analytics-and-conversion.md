# Analytics and Conversion

Use this playbook for public-facing websites, marketing surfaces, SaaS onboarding flows, lead-generation sites, and content systems.

## Default Rule

Analytics and conversion tracking are default layers on public surfaces. Do not ship without a clear idea of what success looks like and which events matter.

## Define Before Release

Identify:

- primary conversion event
- secondary engagement events
- CTA click events
- form start and form success events
- content-to-conversion paths
- important outbound actions such as call, WhatsApp, booking, demo, or checkout

## Tracking Principles

- instrument only what is meaningful
- align events with actual business goals
- avoid noisy or vanity-only tracking
- make sure tracked actions map to visible user flows
- consider privacy and consent requirements before recommending tracking

## Content and SEO Connection

For content-led sites, define:

- which content clusters support which conversion pages
- which articles should drive readers toward which CTA
- which FAQ or answer surfaces support trust and conversion

## Implementation Note

If no dedicated analytics skill exists in the installed library, express the analytics requirements clearly and route implementation through the active engineering and review skills for the project.
